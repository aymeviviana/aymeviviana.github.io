Classes vs. Modules

04.02.2015

What Is A Class?

We’re already familiar with classes from our Hogwarts post. A ruby class is a structure that groups together related methods and variables for the sake of organization and reduced code-redundancy. To complement our knowledge of classes we’ll now learn about Modules.

What Is A Module?

According to Codecademy…“You can think of a module as a toolbox that contains a set of methods and constants.”

Modules are very similar to classes except that modules can’t create instances and can’t have subclasses, they are simply used to store tools.

Module Syntax

Creating a module is very simple:

  1. A module is declared with the keyword “module”
  2. Module names are written in CapitalizedCamelCase

      module ModuleName

        # module code goes here

      end

What Goes In A Module?

Modules commonly store Constants and methods. They tend not to store variables because they have a nasty habit of changing. An example of a ruby constant is PI, which lives in the Math module and is equal to 3.1415….

Why Use Modules?

One of the main purposes of modules is to separate methods and constants with the same name. This is called name-spacing.

For example, if you have the constant PI defined as 3.1415 in the Math module and 3.14 in the Circle module you’ll need a way to differentiate between the two PI’s. Below is an example of name-spacing:


      Math::PI vs. Circle::PI

The double colon tells ruby which Module you’re searching for the PI constant. In the first example we’re looking for PI in the Math module and in the second we are looking for PI in the Circle module.

“Modules define a namespace, a sandbox in which your methods and constants can play without having to worry about being stepped on by other methods and constants.” ruby-doc.com


How Do Classes & Modules Interact?

Any class that includes a certain module can use those module’s methods. You can do this by using the keyword “include” followed by the module name, inside of the class.


      class Hogwarts
        include Math

          def calculate_quidditch_goal_post_circumference(radius)
              @circumference = 2 * PI * radius
          end

      end

Once a module is explicitly included in a class you no longer have to use the double-colon notation, you can just reference the constant directly.
