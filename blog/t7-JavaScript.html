How Different Are Ruby & Javascript?

April 9, 2015

I remember taking the CodeCademy JavaScript track last August. The experience was intesely fulfilling. It was the first programming language I'd ever learned. Of course there were several moments where my brow would suddenly (and aggressively) wrinkle and I felt an overwhelming wave of frustration wash over me. I remember feeling like my brain was crying inside my skull.

Javascript may be the language of the web but it is NOT a good first language to learn when you first pick up programming. However, returning to Javascript after 4 solid weeks of drilling Ruby made it significantly easier understand everything that CodeCademy track was trying to teach me.

They're Not All That Different After All

All the concepts are the same. If you know what a loop does in Ruby, you know what it does in JS. If you know how control flow works in Ruby, then JS won't have many surprises. Ruby data structures like arrays and hashes have their Javascript equivalents, and they look exactly the same.

The main differences I've encountered between the two languages fall into the following three areas: syntax, class structure and built-in methods/functions.

JavaScript Is Heavy On The Syntax

JS's syntax rules rank a lot higher on the high-maintenance scale than Ruby's. My personal favorites include the mandatory semicolon at the end of each statement, the parentheses and curly braces that accompany if statements and the usage of the keyword "var" before every new variable declaration.

You can take a quick look at some key syntactical differences between the two languages on AgentCooper's cheat sheet. The side-by-side comparison does a great job of demonstrating significant overlap between the two languages, with the main difference being the syntactical demands.

For example, take a look at the way the .each method is implemented in Ruby vs. Javascript. Both chunks of code below function the same way and return the same value, but one requires a few more key strokes than the other:

Ruby:
array = [1, 2, 3, 4]
array.each { |x| puts x }

JavaScript:
var array = [1, 2, 3, 4];
array.forEach(function(x) {
  console.log(x);
});

No Classes For Javascript

Nope. None. The last four weeks of Ruby felt like they were building up to the big reveal that is Ruby classes. Then we dove into Javascript and classes are nowhere to be found. You can, however, use object constructors to simulate Ruby-like, class behavior. Here's some good reading on the subject: 3 Ways To Define A Javascript Class.

Where Are The Enumerables?

Finally, there's a surprising lack of magically powerful enumerable methods to do all the heavy lifting for me when programming in Javascript. I must say, I miss all of my amazing enumerables. Writing out a for loop is not nearly as fun. That said, after 4 solid weeks of practice with Ruby iterators, JavaScript loops don't seem as scary or daunting as they did last August.

In conclusion, Javascript is not that bad at all, and I hear it only gets better from here.
