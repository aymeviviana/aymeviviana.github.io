Ruby Classes For Hogwarts!

March 26, 2015

Ruby Classes are especially useful when solving really, really big problems. The larger the scale of the problem, the more you find yourself needing order. Ruby classes provide this organization by grouping together related methods which helps to reduce the occurrence of redundant code.

If you've ever found yourself in a situation where you've written 5 different methods that basically do the same thing, establishing a ruby class can help streamline your code.

A Close Look At Ruby Class Structure

Let's say you were just offered the extremely coveted role of Logistics Administrator at the prestigious Hogwarts School of Witchcraft and Wizardry (you lucky, lucky bastard). You are now in charge of making sure the world's premiere magic institution runs smoothly. It's a HUGE responsibility and you have no interest in hufflepuffing-it-up. It's time to set up a ruby class.

You start by naming your class Hogwarts, because all of your logistical management responsibilities exist within the realm of the Hogwarts school

class Hogwarts

end

Since you'll be in charge of things like drawing up student class schedules and sending out report cards you'll need to gather some information about every student and create processes that will streamline your various responsibilities.

Each new student will be an "instance" of the Hogwarts class and will provide 3 pieces of information for your database of student information which you will store in your "initialize" method. The three pieces of information include: first name, Wizard/Witch orientation and magical specialty.

class Hogwarts
  def initialize(name, orientation, specialty)
    @name = name
    @orientation = orientation
    @specialty = specialty
  end
end

harry_potter = Hogwarts.new("Harry", "Wizard", "Undecided")
hermione_granger = Hogwarts.new("Hermione", "Witch", "Expert at everything")
ronald_weasley = Hogwarts.new("Ron", "Wizard", "Conjuring Slugs")

Now that you've gathered information about every student attending Hogwarts, you want to get started operationalizing some high priority processes that'll get the school up and running

Hogwarts' Finest Instance Methods

First things first! We must sort each student into their respective Hogwarts house with none other than the Sorting Hat! Later on in the year we'll need to finalize the Quidditch rosters for the upcoming Quidditch World Cup and send out the grades reflecting each student's performance in Snape's Potions class.

Since we'll need to evaluate each student along a specific set of metrics to complete each of these tasks we will set up instance methods that will do all the work for us.

Each of these methods contains variables, code blocks and algorithms that sort students into houses, evaluate their lieklyhood of making the Quidditch team, and compute their Potions grade. To see the results for each student all we need to do is call the repective method on the student's name.

hermione_granger.potions_grade
=> "A+++"

harry_potter.sorting_hat
=> "Gryffindor"

ronald_weasley.quidditch_roster

All of these methods can be recycled among different instances of the Hogwarts class or, in this case, to any new student. So with a little bit of organization we've managed to streamline 3 important Hogwarts processes that will benefit thousands of students.
